## Verdict

- openMSX (reference) and this emulator (new model) BOTH show a first-DRQ latency that VARIES
  with the rotational start phase across a ~1-rotation band -- the behavior DEC-0053 R-A required.
- Bounded-consistent, NOT byte-identical: absolute per-phase values differ because openMSX locks
  onto real raw-track addrIdx positions (RealDrive.cc:453 getNextSector) + gapLength
  (WD2793.cc:624 startReadSector), while our CHS model uses 9 evenly-spaced sequential sectors
  (DiskDrive::cycles_until_sector_id) + a fixed 47-byte ID-header span. The variance + the
  0..~1-rotation magnitude band match.
- Old model (pre-slice-C): drq_deadline_ = t + kReadStartCycles == t + 228 for EVERY phase --
  a FLAT, zero-variance latency. That is the defect this slice corrects.
